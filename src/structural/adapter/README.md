# Патерн Адаптер (Adapter Pattern)

## Що це таке?

Патерн Адаптер перетворює інтерфейс одного класу в інший інтерфейс, якого очікують клієнти. Адаптер дозволяє класам з несумісними інтерфейсами працювати разом.

## Коли використовувати?

- Коли хочете використовувати існуючий клас, але його інтерфейс не відповідає потребам
- Коли потрібно створити повторно використовуваний клас, що співпрацює з несумісними класами
- Коли хочете адаптувати кілька існуючих підкласів, але не зручно змінювати їх інтерфейси

## Структура

1. **Клієнт (Client)** - клас, що використовує цільовий інтерфейс
2. **Цільовий інтерфейс (Target)** - інтерфейс, якого очікує клієнт
3. **Адаптований (Adaptee)** - клас з несумісним інтерфейсом
4. **Адаптер (Adapter)** - клас, що адаптує інтерфейс Adaptee до Target

## Приклад використання

```typescript
// Клієнт очікує AmericanPlug інтерфейс
function usePlug(plug: AmericanPlug) {
  console.log(plug.plugIn());
}

// У нас є EuropeanSocket з іншим інтерфейсом
const europeanSocket = new EuropeanSocket();

// Використовуємо адаптер для сумісності
const adapter = new Adapter(europeanSocket);
usePlug(adapter); // Тепер працює!
```

## Переваги

- **Сумісність**: Дозволяє несумісним інтерфейсам працювати разом
- **Принцип єдиного обов'язку**: Адаптер відповідає тільки за адаптацію
- **Відкритий принцип**: Легко додавати нові адаптери для нових інтерфейсів
- **Не змінює існуючий код**: Не потрібно змінювати адаптований клас

## Недоліки

- Збільшує складність системи додатковими класами
- Іноді краще перепроектувати інтерфейси замість використання адаптера
