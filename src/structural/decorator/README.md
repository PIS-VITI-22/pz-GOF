# Патерн Декоратор (Decorator Pattern)

## Що це таке?

Патерн Декоратор динамічно додає додаткові обов'язки до об'єкта. Він забезпечує гнучку альтернативу наслідуванню для розширення функціональності.

## Коли використовувати?

- Коли потрібно додавати обов'язки до окремих об'єктів динамічно та прозоро
- Коли розширення через наслідування незручне або неможливе
- Коли потрібно додавати кілька обов'язків, що можуть комбінуватися
- Коли обов'язки можуть додаватися та видалятися під час виконання

## Структура

1. **Компонент (Component)** - інтерфейс для об'єктів, що можуть мати додаткові обов'язки
2. **Конкретний компонент (Concrete Component)** - клас, до якого додаються обов'язки
3. **Декоратор (Decorator)** - абстрактний клас, що зберігає посилання на компонент
4. **Конкретний декоратор (Concrete Decorator)** - додає конкретні обов'язки до компонента

## Приклад використання

```typescript
// Почати з базової кави
let coffee: Coffee = new SimpleCoffee();
console.log(`${coffee.description()}: $${coffee.cost()}`);

// Динамічно додавати декоратори
coffee = new MilkDecorator(coffee);
coffee = new SugarDecorator(coffee);
console.log(`${coffee.description()}: $${coffee.cost()}`);
```

## Переваги

- **Гнучкість**: Додає обов'язки динамічно без зміни існуючого коду
- **Принцип єдиного обов'язку**: Кожен декоратор має одну відповідальність
- **Композиція над наслідуванням**: Уникає вибухового зростання підкласів
- **Відкритий принцип**: Легко додавати нові декоратори

## Недоліки

- Багато маленьких об'єктів можуть ускладнити налагодження
- Послідовність декорування може впливати на результат
- Важко видалити конкретний декоратор з ланцюжка
