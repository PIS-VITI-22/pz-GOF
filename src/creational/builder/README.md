# Патерн Будівельник (Builder Pattern)

## Що це таке?

Патерн Будівельник відокремлює конструювання складного об'єкта від його представлення, дозволяючи створювати різні представлення одного й того самого процесу конструювання.

## Коли використовувати?

- Коли процес створення об'єкта включає багато кроків
- Коли потрібно створювати різні представлення об'єкта
- Коли конструктор з великою кількістю параметрів стає незручним
- Коли хочете забезпечити незмінність об'єкта після створення

## Структура

1. **Продукт (Product)** - складний об'єкт, що створюється
2. **Будівельник (Builder)** - абстрактний інтерфейс для створення продукту
3. **Конкретний будівельник (Concrete Builder)** - реалізація будівельника
4. **Директор (Director)** - керує процесом створення за допомогою будівельника
5. **Клієнт (Client)** - використовує директора для створення продукту

## Приклад використання

```typescript
// Створення будівельника та директора
const builder = new MargheritaPizzaBuilder();
const director = new PizzaDirector(builder);

// Конструювання піци через покроковий процес
const pizza = director.constructPizza();
console.log(pizza.getDescription());
```

## Переваги

- **Контроль процесу створення**: Покрокове конструювання складних об'єктів
- **Гнучкість**: Можна створювати різні представлення одного продукту
- **Читабельність**: Код створення більш зрозумілий, ніж конструктори з багатьма параметрами
- **Незмінність**: Продукт можна зробити незмінним після створення

## Недоліки

- Збільшує кількість класів у системі
- Будівельник та продукт тісно зв'язані
- Не підходить для простих об'єктів
