# Патерн Фабричний Метод (Factory Method Pattern)

## Що це таке?

Патерн Фабричний Метод визначає інтерфейс для створення об'єктів, але дозволяє підкласам вирішувати, який клас інстанціювати. Фабричний метод передає відповідальність за створення об'єктів підкласам.

## Коли використовувати?

- Коли клас не може передбачити, об'єкти якого типу йому потрібно створити
- Коли хочете делегувати створення об'єктів підкласам
- Коли створення об'єктів вимагає складної логіки ініціалізації
- Коли хочете уникнути зв'язності між класами, що створюють об'єкти

## Структура

1. **Продукт (Product)** - інтерфейс або абстрактний клас створюваних об'єктів
2. **Конкретні продукти (Concrete Products)** - реалізації продукту
3. **Творець (Creator)** - абстрактний клас з фабричним методом
4. **Конкретні творці (Concrete Creators)** - реалізації творця, що перевизначають фабричний метод

## Приклад використання

```typescript
// Створення фабрик для різних типів тварин
const dogFactory = new DogFactory();
const catFactory = new CatFactory();

// Отримання звуків через фабричний метод
console.log('Собака каже:', dogFactory.getAnimalSound());
console.log('Кіт каже:', catFactory.getAnimalSound());
```

## Переваги

- **Принцип єдиного обов'язку**: Розділяє створення об'єктів від їх використання
- **Принцип відкритий/закритий**: Легко додавати нові продукти через нові фабрики
- **Зв'язність**: Клієнтський код не залежить від конкретних класів продуктів
- **Розширюваність**: Легко додавати нові типи продуктів

## Недоліки

- Може призвести до великої кількості класів (по одному на продукт)
- Складніше для простих випадків, де достатньо простого конструктора
