# Патерн Стратегія (Strategy Pattern)

## Що це таке?

Патерн Стратегія визначає сімейство алгоритмів, інкапсулює кожен з них і робить їх взаємозамінними. Він дозволяє вибирати алгоритм під час виконання програми.

## Коли використовувати?

- Коли у вас є кілька алгоритмів, які роблять одне й те саме, але по-різному
- Коли потрібно вибирати алгоритм динамічно під час виконання
- Коли хочете уникнути великої кількості умовних операторів (if-else)
- Коли алгоритми можуть змінюватися незалежно від клієнтського коду

## Структура

1. **Стратегія (Strategy)** - інтерфейс, що визначає метод для виконання алгоритму
2. **Конкретні стратегії (Concrete Strategies)** - реалізації різних алгоритмів
3. **Контекст (Context)** - клас, що використовує стратегію і делегує їй роботу

## Приклад використання

```typescript
// Створення сортувальника з різними стратегіями
const sorter = new Sorter(new BubbleSort());
console.log('BubbleSort:', sorter.sortData([3, 1, 4, 1, 5]));

// Зміна стратегії під час виконання
sorter.setStrategy(new QuickSort());
console.log('QuickSort:', sorter.sortData([3, 1, 4, 1, 5]));
```

## Переваги

- **Відкритий принцип (Open/Closed)**: Легко додавати нові стратегії без зміни існуючого коду
- **Принцип єдиного обов'язку**: Кожна стратегія відповідає за один алгоритм
- **Гнучкість**: Можна змінювати поведінку під час виконання
- **Тестування**: Легко тестувати кожну стратегію окремо

## Недоліки

- Збільшує кількість класів у системі
- Клієнт повинен знати про різні стратегії та їх призначення
