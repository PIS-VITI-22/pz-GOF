# ₴pz-GOF

Проект з прикладами патернів GOF (Factory Method, Builder, Singleton, Adapter, Facade, Decorator, Strategy, Observer).

* **Singleton:** Вирішує проблему надмірного споживання пам'яті, гарантуючи лише одну точку підключення до ресурсів.
* **Factory Method:** Дозволяє системі бути відкритою для розширення (Open/Closed Principle). Легко додати "SMS-сповіщення", не змінюючи код головного контролера.
* **Facade:** Зменшує зв'язність коду. Оператор просто викликає `activate()`, не знаючи про внутрішні налаштування GPS чи двигуна.
* **Strategy:** Покращує архітектуру, замінюючи складні `if/else` на окремі класи-алгоритми, які можна тестувати ізольовано.
* **Adapter:** Дозволяє використовувати сучасних роботів зі старими пультами керування або сенсорами.

**Як запустити:** Використовуйте команду:      `npm start
